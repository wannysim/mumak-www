같은 셔터를 누르고도 **카메라가 저장하는 정보량은 형식에 따라 수십 배 차이**가 난다.

## 핵심 비교

|                | RAW                           | JPEG                    |
| -------------- | ----------------------------- | ----------------------- |
| 비트뎁스       | 12 / 14 / 16 bit              | **8 bit 고정**          |
| 채널당 단계 수 | 4096 / 16384 / 65536          | **256**                 |
| WB             | 후보정에서 자유롭게 변경 가능 | 베이크됨 (변경 시 손실) |
| 색공간         | 센서의 원본 색역              | sRGB / AdobeRGB로 압축  |
| 파일 크기      | 20~100MB                      | 2~10MB                  |
| Lightroom 호환 | 모든 슬라이더 강력 동작       | 슬라이더 효과 약함      |

## 비트뎁스 — 색의 단계 수

8bit = 채널당 **2⁸ = 256단계**.
14bit = 채널당 **2¹⁴ = 16384단계**.

RGB 합치면:

- 8bit: 256³ ≈ **1670만 색**
- 14bit: 16384³ ≈ **4.4조 색**

수치만 보면 8bit도 충분해 보이지만, **보정을 가하는 순간 차이가 폭발**한다.

## JPEG가 보정에서 죽는 이유 — 밴딩(Banding)

256단계 안에서 그라데이션이 만들어지는데, 보정으로 일부 영역을 늘리면(예: 어두운 영역 +50%) **그 영역의 단계 수는 그대로지만 더 넓은 톤에 펴진다** → 단계 사이가 보일 정도로 떨어짐.

```
JPEG 원본:  ▓▓▒▒░░ (6단계로 그라데이션)
+50% 들어올리면: ▓ ▓ ▒ ▒ ░ ░ (단계 사이 빈자리)
                  ↑ 하늘에 줄무늬가 보이는 이유
```

본인이 경험한 "JPEG로 찍은 사진의 암부/명부가 살아나지 않음"이 정확히 이 현상. 8bit는 처음 한 번의 보정 정도까지는 버티지만, 강하게 끌어올리면 무너진다.

## 12bit vs 14bit RAW

카메라 메뉴에서 본 적 있는 옵션. 차이:

- **고감도 / 저조도**: 14bit가 우세. 어두운 영역에 더 많은 단계 → 노이즈 보정 후에도 부드러움.
- **연사 속도**: 14bit는 파일이 커서 카메라 버퍼가 빨리 찬다. 스포츠 촬영자는 12bit 선택하기도.
- **압축 RAW**: Sony의 "Lossless Compressed RAW" 같은 옵션이 14bit를 유지하면서 파일을 줄여줌. 대부분 이걸 켜는 게 정답.

## RAW가 정확히 무엇을 보존하는가

JPEG는 **카메라가 해석을 끝낸 결과물**. RAW는 **센서가 받은 신호 그 자체**.

- WB가 "기록되지 않고" 메타데이터로만 따라온다 → 후보정에서 자유 변경
- 픽처 프로파일 / 색감도 메타데이터 → 변경 자유
- 노출은 빛 자체의 양이라 RAW도 변경 불가 (촬영 시 정해짐)
- 셔터·조리개·ISO도 변경 불가

→ **노출과 ISO는 촬영 시점에 정해야 하지만, 색·톤은 RAW면 거의 다 후보정 가능**.

## "RAW로 찍으면 보정 안 해도 된다"는 오해

RAW는 **색감이 입혀지지 않은 평면적인 상태**다. 카메라 LCD에서 보이는 예쁜 색은 JPEG 미리보기. Lightroom에서 RAW를 열면 처음엔 더 밍밍하게 보일 수 있다. **보정을 해야 비로소 RAW의 가치가 드러난다**.

## 실전 권장

- **항상 RAW로 찍기** (또는 RAW + JPEG)
- Sony 카메라면 **Lossless Compressed RAW**, Canon이면 **CRAW**, Nikon은 **Compressed RAW** 활용. 화질 손실 없이 파일 절반.
- **JPEG만 찍어야 하는 상황**(SNS 즉시 업로드 등)에서도 가능하면 **RAW + JPEG** 동시 저장. 만약 좋은 사진이 나오면 RAW가 보험.
