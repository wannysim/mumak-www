사진의 **밝기 분포 그래프**. Lightroom 우측 상단의 그 작은 그래프. 카메라 LCD에서도 RAW 노출 판단의 가장 정확한 지표.

## 그래프 읽는 법

```
픽셀 수
↑
│       ╭──╮
│     ╭─╯  ╰─╮
│   ╭─╯       ╰╮
│ ╭─╯           ╰─╮
└─┴───────────────┴───→ 밝기
  0                255
  검정             흰색
```

- **X축**: 어두움(0) ↔ 밝음(255)
- **Y축**: 그 밝기에 해당하는 **픽셀의 양**
- **봉우리 위치**: 사진의 톤이 어디에 몰려 있는가

## 4개 영역과 슬라이더 매핑

[[dynamic-range]]의 4개 슬라이더가 히스토그램 영역과 일대일 대응한다.

```
┌──────────────────────────────────────────────────┐
│ Blacks │ Shadows │ ★Mid★ │ Highlights │ Whites  │
│  0~10  │  10~30  │ 30~70 │   70~90    │ 90~100  │
└──────────────────────────────────────────────────┘
   왼쪽 끝 ─────────────────────────────── 오른쪽 끝
```

Lightroom 히스토그램 위에 마우스를 올리면 **각 영역이 하이라이트되며 어떤 슬라이더에 대응하는지** 보여준다. 이 기능 하나만 익혀도 보정이 정밀해진다.

## 클리핑(Clipping) 경고

히스토그램 양 끝 위쪽의 **작은 삼각형**.

- **왼쪽 삼각형 점등(파랑)**: 검정 영역에서 디테일이 잘림. 어둡게 죽은 픽셀 존재.
- **오른쪽 삼각형 점등(빨강)**: 흰색 영역에서 디테일이 잘림. 하얗게 날아간 픽셀 존재.

삼각형을 **클릭**하면 사진 위에 **클리핑 영역이 색으로 오버레이**된다. 어디가 죽었는지 정확히 보임.

> **단축키 J**: 클리핑 오버레이 토글. 보정 중 자주 누르게 됨.

## 노출이 잘 잡힌 사진의 히스토그램?

**정해진 답이 없다**. 장면 자체가 어두우면 히스토그램이 왼쪽으로 쏠리는 게 정상이고, 눈 풍경은 오른쪽으로 쏠려야 정상.

하지만 **권장 원칙**:

- **양 끝까지 데이터가 닿되, 클리핑은 피한다** — 히스토그램의 양쪽이 완전히 0인 채로 끝나면 사진이 평면적. 양쪽 끝까지 살짝씩 닿으면 풍부함.
- **봉우리가 한쪽 절벽에 붙어있으면 위험** — 오른쪽 절벽에 붙어 있으면(highlights clipping) 후보정으로 살릴 데이터가 없음.

## "Expose to the Right" — RAW 촬영의 핵심 전략

> **RAW로 찍을 땐 클리핑 직전까지 노출을 오른쪽으로 미는 게 유리하다.**

이유: 디지털 센서는 **밝은 영역에 더 많은 단계 수**를 할당한다. 어두운 영역에서 끌어올리면 노이즈가 따라오지만, 밝은 영역에서 어둡게 내리는 건 거의 손실 없음.

```
센서가 기록하는 톤 단계 (14bit 기준)
가장 밝은 stop:  8192 단계  ←─── 정보가 풍부
다음 stop:       4096 단계
다음 stop:       2048 단계
다음 stop:       1024 단계
가장 어두운 stop: 64 단계   ←─── 정보가 빈약 (노이즈만)
```

→ **촬영 시 살짝 밝게(클리핑 직전)** → **후보정에서 살짝 어둡게**.
이러면 어두운 영역의 노이즈가 줄어든다.

단, JPEG 출력에서는 무의미. RAW + 후보정 워크플로우에서만 의미 있음.

## RGB 분리 히스토그램

Lightroom 히스토그램은 기본적으로 **빨강/녹색/파랑 채널을 겹쳐서** 보여준다. 클릭하면 채널별로 분리해서 볼 수 있다.

- **빨강만 클리핑** → 하늘에 노을이 너무 진하거나, 인물 피부가 너무 빨갛게 탔을 가능성
- **파랑만 어두운 끝에 몰림** → 그림자에 파랑이 부족 → 따뜻한 톤
- **세 채널이 같은 위치에 봉우리** → 무채색에 가까운 사진

## 카메라 LCD의 히스토그램이 정확하지 않은 이유

촬영 시 카메라가 보여주는 히스토그램은 **JPEG 미리보기 기준**이지 **RAW 기준이 아니다**.

- JPEG에서는 클리핑처럼 보여도 RAW에는 디테일이 살아 있을 수 있음 (≈ 1 stop 여유).
- 그래서 LCD 히스토그램에서 살짝 클리핑 경고가 떠도 RAW면 후보정 가능.
- 정확한 판단은 **Lightroom으로 가져와서 RAW 히스토그램**으로.

## 실전 보정 순서 with 히스토그램

1. **히스토그램 펼쳐 놓고** Exposure 슬라이더로 봉우리를 적정 위치로
2. **양 끝 클리핑 삼각형 확인** → 빨강·파랑 떠 있으면 J 눌러 영역 확인
3. **Highlights / Shadows로 양 끝 회수**
4. **Whites / Blacks로 끝점 정의** → 양쪽 절벽까지 살짝 닿게
5. 톤이 잡히면 [[white-balance|WB]], [[hsl-and-color-mixer|HSL]], [[tone-curve|커브]]로 색 작업
